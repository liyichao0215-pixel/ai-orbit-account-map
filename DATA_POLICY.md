# Data Policy

## 公开演示数据

本仓库包含一个用于产品原型学习的关系图快照：

- 16 个官方 AI 产品账号保留公开资料；
- 脱敏前最终确定的 44 个 S 级和 81 个 A 级候选保留公开资料；
- 其余 B、C、WATCH 账号使用匿名编号，移除真实身份、头像和外链，并扰动精确指标；
- 关注拓扑被保留，因此它仍可能包含关系层面的推断风险，不应被用于身份还原。

## 分级与脱敏顺序

```text
授权源数据
→ 计算候选分级
→ 锁定最终等级
→ 按等级执行公开 / 脱敏策略
→ 公开前端只读取锁定等级
```

匿名名称、匿名简介和扰动指标不得再次用于生成 S/A 结论。当前规则版本为 `outreach-v1.1`，隐私转换版本为 `hybrid-public-priority-v1.1`。

## 使用边界

- 被官号关注不等于品牌背书、合作关系或联系许可；
- 评分是产品假设，只用于候选初筛；
- 快照不是实时数据；
- 不得使用本项目自动私信、自动关注或批量触达；
- 不得尝试恢复匿名账号身份；
- 如需换成自有数据，只能使用你有权读取、存储和处理的导出或接口。

## 重新生成

刷新脚本不内置第三方源站。必须显式传入你有权访问的 URL：

```bash
pnpm reload:snapshot -- "https://example.com/authorized-graph.json"
```

原始未脱敏文件不得放入 `public/`，不得提交到 Git。

## 删除与纠错

如果公开账号资料存在错误，或账号所有者希望移除，请在 GitHub 创建 issue，并提供对应公开账号与理由。维护者应优先删除或更正争议信息。
