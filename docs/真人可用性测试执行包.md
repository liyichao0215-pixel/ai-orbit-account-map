# AI Orbit 真人可用性测试执行包

> 状态：待执行。这个文件是招募、主持、观察和记录模板，不是已经完成的真人研究报告。

## 1. 要验证的核心问题

真实矩阵运营或达人运营能否在 5 分钟内：

1. 说清这个产品解决什么问题；
2. 找到 3 个公开 S/A 候选；
3. 解释至少 1 个候选为什么值得继续人工核验；
4. 把候选加入清单并写回负责人、状态和下一步；
5. 正确说出数据与评分的限制。

## 2. 建议样本

招募 3–5 位真实使用者，建议覆盖：

- 1 位矩阵号运营负责人；
- 1 位达人 / 创作者生态运营；
- 1–2 位一线运营；
- 1 位刚入职或经验较少的运营实习生。

AI Agent 的角色模拟只能用作上线前预检，不计入真人样本。

## 3. 5 分钟任务脚本

主持人只读任务，不解释按钮：

> 你正在为 AI 视频矩阵寻找值得进一步研究的创作者。请先判断这个页面能帮你做什么，然后找到一个你认为值得优先研究的账号，说明理由，把它加入运营清单，填写负责人、状态和下一步动作。最后告诉我：这个页面有哪些信息不能直接相信或仍需人工核验。

如果参与者停顿超过 45 秒，只问：“你现在预期在哪里找到这项信息？”不要直接告诉答案。

## 4. 观察记录

| 指标 | 记录方式 | 目标 |
|---|---|---:|
| 首次说对产品用途 | 是否在 60 秒内提到“关系发现 / 候选初筛” | ≥ 80% |
| 找到公开 S/A 候选 | 是否完成筛选或通过雷达找到候选 | ≥ 80% |
| 解释候选证据 | 是否提到共同关注，并至少提到另一项评分因素 | ≥ 80% |
| 完成清单写回 | 是否填入负责人、状态、下一步 | ≥ 80% |
| 识别边界 | 是否说出“快照 / 非实时 / 需人工核验”任意两项 | ≥ 60% |
| 任务时间 | 从读完任务到完成写回 | 中位数 ≤ 5 分钟 |

同时记录：

- 第一次点击的位置；
- 误以为可以自动完成的动作；
- 看不懂的术语；
- 3D 图是否帮助发现，还是造成迷失；
- 放弃或返回的节点；
- 参与者的原话（需获得记录同意）。

## 6. 伦理与隐私

- 测试前说明这是个人作品集原型，不是公司正式系统；
- 参与者可随时停止；
- 未经同意不录屏、不录音、不公开姓名或公司；
- 测试中不要输入真实联系方式、报价、私信内容或未公开合作信息；
- 对外报告只使用匿名编号和汇总结论。

## 7. 结果判定

- 若 3–5 人中至少 80% 完成候选发现与清单写回：可以继续验证长期使用；
- 若多数人只使用列表、很少使用 3D：保留 3D 作为探索入口，默认工作区改为二维；
- 若评分解释经常被理解成自动结论：进一步弱化等级，增加“人工核验清单”语言；
- 若用户不愿维护状态：停止扩建 CRM，回到纯关系研究工具。

对外只能写“已完成真人测试”的前提是：确实招募了真人、保存了匿名记录，并能说明样本与方法。
