# PRD：账号关系可视化看板（学习版）

## 1. 产品目标

帮助矩阵账号运营者从多个核心账号的一跳关注网络中发现共同关注对象、关系社区和候选账号，并获得可解释的优先级依据。

## 2. MVP 范围

包含：分层脱敏快照导入、3D 社区图、搜索、分类筛选、节点详情、共同关注解释、候选雷达、本机运营清单、CSV 导出、响应式布局。

不包含：第三方平台登录、实时抓取、私信发送、自动关注、多人协作。官号和最终 S/A 候选保留其原本已公开的账号资料；B/C/WATCH 账号移除真实身份与外链。

## 3. 关键流程

```text
进入看板 → 浏览关系社区 → 搜索或筛选 → 点击节点 → 查看关系与评分 → 加入运营清单 → 写回负责人 / 状态 / 备注 → 导出
```

## 4. 信息优先级

1. 当前看的是哪个范围：全部、优先、核心官号或产品类别。
2. 账号位于哪个关系社区，与哪些核心号有关。
3. 候选为什么得到当前评分。
4. 当前数据是否脱敏、是否实时、是否可以外跳。

## 5. 风险与保护

- 数据风险：官号和最终 S/A 级候选仅保留公开资料；B 级及以下账号必须移除身份信息与外部链接，并扰动精确指标。公开发布时还应使用合成拓扑，避免关系结构带来再识别风险。
- 解释风险：评分是启发式规则，界面不得暗示为 AI 的客观结论。
- 视觉风险：3D 遮挡可能影响查找，因此必须保留搜索、筛选和列表。
- 合规风险：未来接入真实平台数据前，需确认账号授权、接口条款和保存周期。

## 6. 验收标准

- 本地可启动，且不依赖真实头像或主页链接。
- 16 个公开官号、125 个公开 S/A 候选、1,668 个匿名节点和 2,410 条关系可加载。
- 官号及 S/A 公开资料可显示；B 级及以下账号的真实名称、ID、头像 URL、主页 URL 不出现在公开 JSON 中。
- 用户可以搜索官号、S/A 账号或匿名账号，按类别筛选、点击节点并查看关系。
- 用户可以打开内置的 5 步产品拆解。
- 桌面与移动端均不阻断核心流程。

## 7. V2 当前实现与下一假设

当前已实现设备本地的运营清单，包括负责人、状态、备注、下一步动作和 CSV 导出，用于验证“关系发现之后能否继续推进工作”。

下一步假设是：如果 3–5 位真实运营者在任务测试中愿意持续写回状态，再考虑账号登录、多人协作、提醒和业务结果归因；在此之前不建设完整 CRM。
